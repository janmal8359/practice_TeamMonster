
 1.  _WAIT 상태 행동

첫등장시 _WAIT 상태행동을 적용
(_WAIT의 기본 상태는 n초동안의 인터벌을 가지고 캐릭 반대편인 x/2 +200정도에서 왼쪽 idle을 가지고 있음)
해당 _WAIT의 상태는 보스의 다른 상태행동에 영향을 가지며 각각 엘보우, 메테오, 스매시, 기상어택, 하울링어택등 
다른 상태행동 전과 후에 n초동안의 인터벌을 부여해주는 형식으로 사용됨
(조금 더 디테일하게는 _WAIT의 인터벌을 2개를 부여하여 행동후의 인터벌이 긴 상태를 부여해도 괜찮을듯)

 2. _MOVE 상태 행동 (_SLAB ATTACK, ELBOW ATTACK)

 보스의 _MOVE는 겟앵글을 통해서 플레이어 방향으로 이동하며 y축이 플레이어와 같고, 플레이어 x축+- n지점에 있을때
 _WAIT상태에돌입 후 슬랩 어택 한번, 이후에는 엘보우, 메테오어택 순으로 공격 이때 각 엘보우와 메테오 행동전 
 _WAIT행동 돌입
  플레이어와 에너미의 지형지물 충돌방식을 기둥에 제한하며 플레이어x축 +- n은 기둥 렉트 width보다 큼
 이에 플레이어와 에너미 사이에 기둥이 있어도 y축이 같고 서로간의 x축이 기둥만큼의 길이가 있어도 _WAIT후 공격행동

 3.  _METEOR 상태 행동

 보스의 메테오 어택은 슬랩과 엘보우 어택 후에 발동 되며 z축을 이용한 점프로 winsizey보다 -n 만큼 점프함
 이 점프동안 n만큼의 인터벌을 주며 스피드가 플레이어보다 빨라짐
 보스의 그림자로 플레이어를 찾아가며 플레이어 x축과 y축의 +- n 만큼 도달시 n만큼의 인터벌을 가진후 떨어짐
 보스의 z축이 기존 z축보다 커질경우 렉트는 멈추며 그림자의 범위 +-n만큼의 충돌처리 반경을 가짐 
(이때 땅 갈라짐 프레임을 그림자x,y축에 생성되게 함) 
 충돌처리시에 보스의 렉트와 기둥의 렉트가 인터섹트할시에 기둥의 렉트는 사라지게 함
 (기둥의 정보중 bool값을 넣어 if문으로 땅갈라짐 생성시 && 인터섹트렉트 일때 bool값을 true로 바꾸어 해당 기둥을
사라지게 함)

 4.  _STAND ATTACK 상태 행동

 보스의 메테오 어택과 동시에 다시 _WAIT 행동을 취하며 기상어택 행동에 돌입함
 (*보스는 따로 기상 모션이 없으며 이를 기상어택이 전부 대체함) 
기상어택은 _WAIT인터벌 후에 그림자를 중점으로 땅갈라짐 효과와 충돌처리 반경을 가진후 일어나는것 까지로 설정

 5. _DASH ATTACK 상태 행동
 
 메테오 어택 후에는 대시 어택 행동을 하며 대시 어택전, 전체 난이도에 따라 _WAIT 행동을 취할지 뺄지 결정함
 보스의 대시 어택은 겟디스텐스로 휘지않는 이동거리를 가지며 해당 지점에 도착시 연속으로 3번 대시 공격 행동을
가질 예정
 (이때 기둥 렉트에 인터섹트 할시, break처리) 

 6. _HOWLING ATTACK 상태 행동

 3번째 대시공격이 끝났다고 bool 값이나 순서진행으로 이를 알렸을때 보스의 하울링 어택을 시전함
 해당 어택은 _WAIT행동을 행하지 않으며 충돌반경은 모든 어택중 가장 넓은 반경의 보스x,y축 +-n을 가짐

 7. _ATTACKED (피격) 상태 행동 (_DOWN)

 보스의 피격상태 행동은 보스의 _WAIT상태일때만 발생되며 대미지가 2회이상 발생시 _DOWN행동을 행함
_DOWN행동일시 이미지 렌더와 _DOWN 행동 정보에 있는 개별 인터벌을 가진후에 기상어택 행동으로 돌입
(_WAIT 행동 상태가 아니며 다른 행동중일때는 _DOWN행동을 하지 않으며 기존의 행동을 계속 이행)

 8. _GROGGY 상태 행동 

 보스의 그로기는 hp바의 x축이 특정 구간일때 모든 행동을 break하고 _GROGGY 행동을 행함
 해당 행동시 _GROGGY 행동만의 인터벌을 가지며 인터벌이 끝남과 동시에 기상어택 행동으로 돌입

 9. _PHASE2 상태 행동
 
 보스의 페이즈2는 hp바의 x축이 특정 구간일때 모든 행동을 break하고 페이즈2 행동을 행함
 해당 행동이 종료됨과 동시에 하울링어택을 시전하며 이후에 공격행동 순번인 슬랩어택이 스매시 어택으로 변경됨

 10. _DEATH 상태 행동
 
 보스의 죽음 행동은 hp바의 right가 left와 같거나 이하일때 행동되며 모든 행동을 break 하고 죽음 행동 실행
